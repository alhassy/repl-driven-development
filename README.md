# Motivation

Whenever reading/refactoring some code, if you can make some of it
self-contained, then you can immediately try it out! No need to
load your entire program; nor copy-paste into an external REPL.  The
benefits of Emacs' built-in <abbr class="tooltip" style="border: none; text-decoration: none;" title="C-x C-e в€· eval-last-sexp<br>Evaluate sexp before point; print value in the echo area.<br>Interactively, EVAL-LAST-SEXP-ARG-INTERNAL is the prefix argument.<br>With a non вЂ-вЂ™ prefix argument, print output into current buffer.<br><br>This commands handles вЂdefvarвЂ™, вЂdefcustomвЂ™ and вЂdeffaceвЂ™ the<br>same way that вЂeval-defunвЂ™ does.&emsp;See the doc string of that<br>function for details.<br><br>Normally, this function truncates long output according to the<br>value of the variables вЂeval-expression-print-lengthвЂ™ and<br>вЂeval-expression-print-levelвЂ™.&emsp;With a prefix argument of zero,<br>however, there is no such truncation.<br>Integer values are printed in several formats (decimal, octal,<br>and hexadecimal).&emsp;When the prefix argument is -1 or the value<br>doesnвЂ™t exceed вЂeval-expression-print-maximum-characterвЂ™, an<br>integer value is also printed as a character of that codepoint.<br><br>If вЂeval-expression-debug-on-errorвЂ™ is non-nil, which is the default,<br>this command arranges for all errors to enter the debugger.<br><br>This function has :around advice: вЂad-Advice-eval-last-sexpвЂ™.<br><br>(fn EVAL-LAST-SEXP-ARG-INTERNAL)"><kbd style="border-color: red">C-x C-e</kbd></abbr> for Lisp, and Lisp's Repl

Since these new REPL commands are just Emacs functions, we can use
several at the time, alternating between them.  For example:

    ;; C-x C-e on the next two lines
    (repl-driven-development [C-x C-t] terminal)
    (repl-driven-development [C-x C-p] python)

    echo Hello... > /tmp/o       # C-x C-t here

    print(open("/tmp/o").read()) # C-x C-p here

    echo ...and bye >> /tmp/o    # C-x C-t again

    print(open("/tmp/o").read()) # C-x C-p again
